# 🚀 Configuration du Système VRP Complet

## 📋 Vue d'ensemble

Le système est maintenant entièrement configuré avec:

1. **Frontend React+TypeScript** - Interface utilisateur
2. **Backend Node.js/Express** - Serveur API
3. **Python Solver** - Moteur d'optimisation VRP

## 🔧 Installation des dépendances Python

Avant de démarrer le serveur, installez les dépendances Python:

```bash
pip install pandas pulp geopy requests matplotlib numpy
```

## 📊 Flux de Travail

### 1. Configurer les Données

- **SpotManager**: Ajouter des lieux (concerts)
- **GearManager**: Sélectionner le matériel pour chaque lieu
- **FleetManager**: Configurer les véhicules disponibles

### 2. Lancer l'Optimisation

- Cliquer sur le bouton **"Lancer l'Optimisation"** dans RouteSummary
- Le système génère 4 solutions:
  - ✅ Équilibré (par défaut)
  - ✅ Économie Véhicules
  - ✅ Rapidité
  - ✅ Distance Minimale

### 3. Afficher les Résultats

- Les solutions s'affichent immédiatement
- Cliquer sur une solution pour voir les détails des routes
- Comparer les autres solutions pour choisir la meilleure

## 🐛 Dépannage

### "Serveur indisponible"
- Vérifier que le backend est lancé (`npm start` dans `/backend`)
- Vérifier l'adresse `http://localhost:5000/api/health`

### "Erreur parsing résultats"
- Vérifier que Python est installé et les dépendances OK
- Vérifier les logs du serveur backend

### "Aucune solution trouvée"
1. Vérifier la capacité des véhicules vs volume total
2. Relâcher les contraintes temporelles
3. Ajouter plus de véhicules

## 📊 Paramètres d'Optimisation

Les 4 solutions testent différents poids:

| Solution | Véhicule | Temps | Distance | But |
|----------|----------|-------|----------|-----|
| Équilibré | 1000 | 10 | 5 | Bon compromis |
| Économie | 5000 | 5 | 2 | Minimiser véhicules |
| Rapidité | 500 | 50 | 5 | Minimiser temps |
| Distance | 500 | 5 | 50 | Minimiser km |

## 🔌 API Endpoints

### POST /api/optimize
Lanc une optimisation

**Body:**
```json
{
  "lieux": [
    {
      "Id_Lieux": 0,
      "Nom": "Lieux",
      "Adresse": "...",
      "lat": 48.8,
      "lon": 2.3,
      "Instruments": "Guitare, Basse",
      "HeureTot": 480,      // 8:00 en minutes
      "HeureConcert": 600,  // Concert à 10:00
      "HeureTard": 1320     // Fermeture à 22:00
    }
  ],
  "instruments": [
    { "Nom": "Guitare", "Volume": 2.5 }
  ],
  "vehicules": [
    {
      "Id_vehicules": 1,
      "Nom": "Van 1",
      "Volume_dispo": 15
    }
  ]
}
```

**Response:**
```json
{
  "success": true,
  "solutions": [
    {
      "id": 1,
      "label": "Équilibré",
      "nb_vehicules": 2,
      "temps_total_min": 240.5,
      "distance_totale_km": 125.3,
      "objectif": 1234.56,
      "details_vehicules": [...]
    }
  ],
  "best_solution": { ... }
}
```

### GET /api/health
Vérifie l'état du serveur

**Response:**
```json
{
  "status": "ok",
  "timestamp": "2026-02-26T..."
}
```

## 📝 Notes

- Les solutions sont générées en ~30-60 secondes selon la complexité
- La limite de timeout est de 5 minutes
- Le solveur utilise PuLP avec CBC
- Les distances sont calculées via OSRM
