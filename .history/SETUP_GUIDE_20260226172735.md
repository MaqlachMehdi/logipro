## 🛠️ Dépannage

### ❌ "No module named 'pandas'"

**Problème:** Les dépendances Python ne sont pas installées.

**Solution:**
```bash
python -m pip install pandas pulp geopy requests matplotlib numpy
```

### ❌ "Serveur indisponible (http://localhost:5000)"

**Problème:** Le backend n'est pas en cours d'exécution.

**Solution:** Lancez le backend dans un terminal:
```bash
cd c:\logipro\backend
npm start
```

### ❌ "python: command not found"

**Problème:** Python n'est pas dans le PATH.

**Solution:**
1. Réinstallez Python en **cochant "Add Python to PATH"**
2. Ou redémarrez votre terminal après l'installation

### ❌ Port 5000 déjà utilisé

**Problème:** Un autre service utilise le port 5000.

**Solution:** Modifier le port dans `backend/server.js`:
```javascript
const PORT = 5001; // Changez le port
```

## 🎯 Workflow complet

1. **Installer** `python -m pip install pandas pulp geopy requests matplotlib numpy`
2. **Démarrer backend** dans Terminal 1: `cd backend && npm start`
3. **Démarrer frontend** dans Terminal 2: `npm run dev`
4. **Ouvrir** `http://localhost:5173`
5. **Ajouter** lieux et véhicules
6. **Sélectionner** stratégie (Équilibré, Économie, Rapidité, Distance)
7. **Cliquer** "Lancer l'Optimisation"
8. **Voir résultats** dans l'interface
9. **Consulter log** dans `logs/optimization_log.txt`

## 🐛 Dépannage

### "Serveur indisponible"
- Vérifier que le backend est lancé (`npm start` dans `/backend`)
- Vérifier l'adresse `http://localhost:5000/api/health`

### "Erreur parsing résultats"
- Vérifier que Python est installé et les dépendances OK
- Vérifier les logs du serveur backend

### "Aucune solution trouvée"
1. Vérifier la capacité des véhicules vs volume total
2. Relâcher les contraintes temporelles
3. Ajouter plus de véhicules

## 📊 Paramètres d'Optimisation

Les 4 solutions testent différents poids:

| Solution | Véhicule | Temps | Distance | But |
|----------|----------|-------|----------|-----|
| Équilibré | 1000 | 10 | 5 | Bon compromis |
| Économie | 5000 | 5 | 2 | Minimiser véhicules |
| Rapidité | 500 | 50 | 5 | Minimiser temps |
| Distance | 500 | 5 | 50 | Minimiser km |

## 🔌 API Endpoints

### POST /api/optimize
Lanc une optimisation

**Body:**
```json
{
  "lieux": [
    {
      "Id_Lieux": 0,
      "Nom": "Lieux",
      "Adresse": "...",
      "lat": 48.8,
      "lon": 2.3,
      "Instruments": "Guitare, Basse",
      "HeureTot": 480,      // 8:00 en minutes
      "HeureConcert": 600,  // Concert à 10:00
      "HeureTard": 1320     // Fermeture à 22:00
    }
  ],
  "instruments": [
    { "Nom": "Guitare", "Volume": 2.5 }
  ],
  "vehicules": [
    {
      "Id_vehicules": 1,
      "Nom": "Van 1",
      "Volume_dispo": 15
    }
  ]
}
```

**Response:**
```json
{
  "success": true,
  "solutions": [
    {
      "id": 1,
      "label": "Équilibré",
      "nb_vehicules": 2,
      "temps_total_min": 240.5,
      "distance_totale_km": 125.3,
      "objectif": 1234.56,
      "details_vehicules": [...]
    }
  ],
  "best_solution": { ... }
}
```

### GET /api/health
Vérifie l'état du serveur

**Response:**
```json
{
  "status": "ok",
  "timestamp": "2026-02-26T..."
}
```

## 📝 Notes

- Les solutions sont générées en ~30-60 secondes selon la complexité
- La limite de timeout est de 5 minutes
- Le solveur utilise PuLP avec CBC
- Les distances sont calculées via OSRM
